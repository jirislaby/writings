\section{Introduction}
In the contemporary world, we are surrounded by devices making our lives
easier. It counts from modern lifts which speak to us to supercomputers
helping to treat cancer such as \emph{IBM Watson}~\cite{watson-cancer}. Today
it happens that most devices are featuring fast processing units allowing
developers to implement arbitrary functions in them. However being written by
humans vastly, sometimes it leads to errant behavior. There were several
approaches how to avoid or find and dismiss errors from programs and over the
past two decades many new techniques appeared.

\subsection{Motivation}
Although many approaches exist, some are not applicable to real software due
to high computation time needs or sometimes even because of inability to read
sources in the offshoot of a language the projects are written in. In this
thesis we aim at mapping and investigation of current techniques used for code
analysis and their application on real, large software projects.

For that purpose, we chose the Linux kernel. It is a large project running on
zillion machines over the world and kernels of operating systems should
generally be error-free, because even a minor error would allow an arbitrary
attacker to obtain administrator privileges on a machine running such a
vulnerable system. There are examples of such errors like
in~\cite{exploit_vmsplice, exploit_proto_ops, exploit_tun}. Hence there is a
call for techniques to possibly eliminate all errors the kernels contain.

Real code; self-contained code; call to be bug-free, otherwise

new techniques or improve the contemporary ones.
