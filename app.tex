  \chapter{Udev files}
  \label{app.udev}
  Commented file listings of udev rules and a script which is called from
  those rules are following.

  \section{udev rules file}
\begin{lstlisting}[language={},label=lst.udevrules,name=lst.udevrules]
KERNEL!="combosix*", GOTO="cesnet_generator_end"(*@\label{lstl.udevrules_1}@*)
ACTION!="add", GOTO="cesnet_generator_end"(*@\label{lstl.udevrules_2}@*)
NAME=="?*", GOTO="cesnet_generator_end"(*@\label{lstl.udevrules_3}@*)
\end{lstlisting}
  Since we want to match only additions of combosix devices, rules at line
  \ref{lstl.udevrules_1} and \ref{lstl.udevrules_2} are needed. To ignore
  cases where the stored name was set yet, the skip rule at line
  \ref{lstl.udevrules_3} is there.
\begin{lstlisting}[name=lst.udevrules]
ENV{CS}="$kernel"
ENV{MATCHPATH}="$devpath"
\end{lstlisting}
  Set up environment variables used in shell script later. These variables are
  filled in directly by \emph{udev} and corresponds to the name suggested by
  kernel or device path (i.e. bus, slot\ldots) respectively.
\begin{lstlisting}[name=lst.udevrules]
IMPORT{PROGRAM}="csdevname" (*@\label{lstl.udevrules_11}@*)

ENV{CS_NEW}=="?*", NAME="$env{CS_NEW}"

LABEL="cesnet_generator_end" (*@\label{lstl.udevrules_12}@*)
\end{lstlisting}
  And finally invoke the script and set up the new name if the script changed
  it. The last step (line \ref{lstl.udevrules_12}) is to define a label where
  the first 3 rules jump to.

  \section{csdevname script}
  Shell script \textbf{csdevname} called from line \ref{lstl.udevrules_11}
  above follows:
\lstset{language=bash} % XXX TODO
\begin{lstlisting}[label=lst.udevscript,name=lst.udevscript]
RULES_FILE='/etc/udev/rules.d/70-cesnet.rules'

. /lib/udev/rule_generator.functions(*@\label{lstl.udevscript_1}@*)
\end{lstlisting}
  This loads udev predefined functions used later. RULES\_FILE is the
  name of file, where we store our persistent naming. Variable name is
  predefined (and used in the script sourced at line \ref{lstl.udevscript_1})
  and must not be changed.
\begin{lstlisting}[name=lst.udevscript]
cs_name_taken() {
	local value="$(find_all_rules 'NAME=' $CS)"
	if [ "$value" ]; then return 0; else return 1; fi
}

find_next_available() {
	raw_find_next_available "$(find_all_rules 'NAME=' "$1")"
}
\end{lstlisting}
  Functions \textbf{cs\_name\_taken} and \textbf{find\_next\_available} are
  for finding available names for new coming device. Both uses functions
  defined in the sourced script from line \ref{lstl.udevscript_1}, that allows
  be them as compact as possible.
\begin{lstlisting}[name=lst.udevscript]
write_rule() {
	local match="$1"
	local name="$2"
	local comment="$3"
	{
		[ "$comment" ] && echo "# $comment"
		echo "SUBSYSTEM==\"combosix\", ACTION==\"add\"$match, NAME=\"$name\""
	} >> $RULES_FILE
}
\end{lstlisting}
  \textbf{write\_rule} takes 3 parameters -- under which condition this rule
  is triggered, static node name and potentially a comment to this rule. The
  result is written to previously defined \textbf{RULES\_FILE}.
\begin{lstlisting}[name=lst.udevscript]
lock_rules_file(*@\label{lstl.udevscript_27}@*)
choose_rules_file(*@\label{lstl.udevscript_29}@*)
\end{lstlisting}
  At lines \ref{lstl.udevscript_27} and \ref{lstl.udevscript_29} we lock up
  the rules file to prevent race
  conditions (if two writers occurred, it would result in unpredictable
  behavior) and decide in which file will be the new rules written (since
  usual location might be still read-only while booting -- \emph{udev} will
  then move the temporary file to the proper directory after it becomes
  writable).
\begin{lstlisting}[name=lst.udevscript]
MATCHPATH=${MATCHPATH%/*/*}
if [ "$MATCHPATH" ]; then
	match="$match, DEVPATH==\"$MATCHPATH/*\""
fi

if [ -z "$match" ]; then
	echo "missing valid match" >&2
	unlock_rules_file
	exit 1
fi
\end{lstlisting}
  Now we define matching rule passed to \textbf{write\_rule} later (line
  \ref{lstl.udevscript_49}). Sanity check of the value is done here and if it
  fails we return error.
\begin{lstlisting}[name=lst.udevscript]
if [ "$CS_NAME" ]; then
	COMMENT="$COMMENT (custom name)"
	if [ "$CS_NAME" != "$CS" ]; then
		CS=$CS_NAME;
		echo "CS_NEW=$CS"
	fi
else
	base=${CS%%[0-9]*}
	if cs_name_taken; then
		CS="$base$(find_next_available "$base[0-9]*")"
		echo "CS_NEW=$CS"
	fi
fi
\end{lstlisting}
  Here the name, which is returned to rules (\textbf{CS\_NEW}), is set. Logic
  is simple, either the one passed by external application is used or a new
  one is found (first unused).
\begin{lstlisting}[name=lst.udevscript]
write_rule "$match" "$CS" "$COMMENT"(*@\label{lstl.udevscript_49}@*)

unlock_rules_file
\end{lstlisting}
  Finally the file with static naming is augmented and unlocked.

  Since this moment, if the device come in the system again later, even in
  other order, system will ever create a node with the same name loaded from
  the rules file.

  \section{Use Case open\_node}
  \begin{CompactEnumerate}
    \item the module's device node is opened
    \item per application descriptor allocation is done including space for
      status structure
    \item if (not allocate\_on\_modprobe and refcount was 0) then call
      \emph{alloc\_rings}
    \item node is mmap-ed (offset 0) -- status
    \item node is mmap-ed with values from previously mapped memory (rx, tx
      spaces) -- note that all buffers are visible as a virtual contiguos
      memory. The only difference between TX and RX mmio space is read/write
      protection
  \end{CompactEnumerate}

  \section{Use Case subscribe}
  \subsection{Precondition}
    Node must be opened yet and close must not to be in progress

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item subscribe\_area ioctl is called with areas bitmap and poll\_thresh;
      both for each direction
    \item APP poll\_thresh and areas are set to the status page
  \end{CompactEnumerate}

  \section{Use Case start}
  \subsection{Precondition}
    Node must be opened yet and close must not to be in progress

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item start ioctl is called
    \item each APP rx\_tail\_ptr and APP rx\_head\_ptr is set to the current
  corresponding HW rx\_tail\_ptr
    \item if (refcount was 0) then the device is started -- enabling of interrupts,
  capture start ...
  \end{CompactEnumerate}

  \section{Use Case wait\_for\_data}

  \subsection{Precondition}
  Node must be opened yet and close must not to be in progress

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item poll is invoked
    \item while:
    \begin{CompactEnumerate}
      \item for all requested rx areas: difference(HW rx\_head\_ptr, APP
        rx\_tail\_ptr) $<$ APP rx\_poll\_thresh and
      \item for all requested tx areas: (tx\_ring\_size - difference(HW
        tx\_head\_ptr, HW tx\_tail\_ptr)) $<$ APP tx\_poll\_thresh
    \end{CompactEnumerate}
      do: the proccess is slept and interrupt scheduled
    \item return status to the application
  \end{CompactEnumerate}

  \subsection{Alternative flow}
  \begin{CompactItemize}
    \item signal is catched while sleeping $\rightarrow$ interrupt waiting and
      return
  \end{CompactItemize}

  \section{Use Case get\_data}
  \subsection{Precondition}
  Node must be opened yet and close must not to be in progress

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item rx\_lock\_data ioctl is invoked with requested areas bitmap
    \item the requested bitmap is logically ANDed with the subscribed bitmap
    \item size is computed as difference (APP rx\_tail\_ptr, HW rx\_head\_ptr)
      for each area until size is zero and this area number is stored
    \item the size (even 0 is valid) is stored into the status page along with
      offset obtained from APP rx\_tail\_ptr
    \item offset + size sum is stored into the right APP rx\_head\_ptr
    \item stored area is returned
  \end{CompactEnumerate}

  \subsection{Alternative flow}
  \begin{CompactItemize}
    \item size is zero -- invalid area is returned
  \end{CompactItemize}

  \section{Use Case release\_data}

  \subsection{Precondition}
  Node must be opened yet and close must not to be in progress

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item rx\_unlock\_data ioctl was invoked
    \item foreach area do
    \begin{CompactEnumerate}
      \item corresponding APP rx\_tail\_ptr is set to the APP rx\_head\_ptr
        value previously stored in get\_data
      \item HW rx\_tail\_ptr is updated to the min(APP rx\_tail\_ptrs $>$ HW
        rx\_tail\_ptr), if there is no such value, then use min(APP
	rx\_tail\_ptrs) of corresponding area pointers. This is needed to
	catch pointers actually higher than others, but still belonging to the
	slowest readers (head usually wraps around and continues from the
	entry number 0).
    \end{CompactEnumerate}
  \end{CompactEnumerate}

  \section{Use Case request\_space}

  \subsection{Precondition}
  Node must be opened yet and close must not to be in progress.
  Application must have this area subscribed for tx.
  TX for requested area is not in progress (only one writer at a time).

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item request\_space ioctl is called with size and area parameters
    \item size is computed as min(max\_tx\_request\_count, requested size,
  tx\_ring\_buffer\_size - difference(HW tx\_head\_ptr, HW tx\_tail ptr))
    \item the size and HW tx\_head\_ptr as offset is stored into APP status\_page
  \end{CompactEnumerate}

  \section{Use Case put\_data}

  \subsection{Precondition}
  Node must be opened yet and close must not to be in progress.
  request\_space was called by this application for area in parameter.

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item put\_data ioctl is invoked with filled size and area where the data was
  filled to
    \item min(size computed in request\_space, supplied size) is added to the
  HW.tx\_head\_ptr
  \end{CompactEnumerate}

  \section{Use Case close\_node}

  \subsection{Precondition}
  open\_node was invoked before this call

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item stop ioctl was called
    \item node is unmap-ed
    \item module's close function is called
    \item if (refcount will be 0)
  a) the device is stopped (it's waited for dma transfers to be
  aborted/finished)
  b) if (allocation on modprobe) then null HW ptrs else call dealloc\_rings
    \item per application descriptor is deallocated
  \end{CompactEnumerate}


  \section{Use Case interrupt}

  \subsection{Precondition}
  Hardware was started and expected to run (i.e. refcount $>$ 0)

  \subsection{Normal flow}
  \begin{CompactEnumerate}
    \item interrupt (timeout or ring buffer pointer reached) is raised -- ring
  buffer is in state we requested
    \item wake applications, which are waiting for this info up
  \end{CompactEnumerate}

  \subsection{Alternative flow}
  \begin{CompactItemize}
    \item missed interrupt -- set up timer (watchdog) to cope with this
  \end{CompactItemize}

  \section{Use Case insert\_module}
  \begin{CompactEnumerate}
    \item modprobe/insmod the module is invoked
    \item if (allocate on modpobe) then call alloc\_rings
  \end{CompactEnumerate}

  Note: 'allocate on modpobe' is set to true by default, but might be
  overriden by module parameter at insertion time

  \section{Use Case remove\_module}

  \begin{CompactEnumerate}
    \item rmmod the module is called
    \item if (allocate on modpobe) then call dealloc\_rings
  \end{CompactEnumerate}


  \section{Use Case alloc\_rings}

  \begin{CompactEnumerate}
    \item alloc\_ring is called
    \item pages for the ring buffer are allocated, each interface has its own
      area in the buffer (each interface has its own DMA controller and a
      ring buffer)
    \item physical addresses are put to the hardware page pointers list -- the
      last descriptor points to the first
  \end{CompactEnumerate}

  \section{Use Case dealloc\_rings}

  \begin{CompactEnumerate}
    \item all ring buffer (DMA) mappings and buffers are unmapped and
      deallocated
  \end{CompactEnumerate}
